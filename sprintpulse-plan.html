🎯 The Approach

Fastest path to a working POC in 36 hours — no CDK, no complex infra, just ship it

Core Decision: Skip AWS CDK entirely. Use AWS Console + plain Lambda zip deploys to get functions running in minutes. Focus 80% of time on product logic, not infrastructure.

✅ What We Build

  • Standup ingestion — web form + transcript paste + audio upload
  • OpenAI-powered transcript parser with speaker separation
  • Analysis engine — 5 detection types
  • GitHub + Jira API integration for say-do gap
  • React dashboard with real data
  • Sprint health score per person

⚠️ What We Mock

  • 2 weeks of pre-loaded historical standup data
  • Jira if access unavailable — mock ticket responses
  • Auth — hardcoded team config, no login
  • Slack/email notifications — just show on dashboard

❌ What We Skip

  • AWS CDK / CloudFormation
  • Bot joining live calls in real-time
  • Multi-tenant team management
  • User authentication / login
  • Production error handling
  • CI/CD pipeline

Backend — AWS Lambda (Node.js)

  • POST /ingest — accepts text/transcript/audio, stores to DynamoDB
  • POST /parse-transcript — calls OpenAI, extracts per-person standups
  • GET /analyze — runs all 5 detections, fetches Git+Jira, scores
  • GET /dashboard — returns full team analysis for React
  • POST /upload-audio — S3 presigned URL + triggers Transcribe

Frontend — React + Vite

  • Dashboard — team health score, member cards, flags
  • Submit Standup — 3 input modes (audio/transcript/manual)
  • Member Detail — standup history timeline, flag breakdown
  • Leaderboard — suspicion scores, trends over sprint
  • Hosted on localhost for demo (or S3 static hosting)

Fastest infra setup (~45 mins): Create 5 Lambda functions in AWS Console → attach API Gateway HTTP API → one DynamoDB table → one S3 bucket for audio files. Done. No CDK, no YAML, no waiting.

🛠️ Tech Stack

Every tool chosen for speed — things the team already knows or can pick up in minutes

⚡ AWS Lambda

Node.js 20. All backend logic. Deploy as zip via console. No servers to manage.

🌐 API Gateway

HTTP API (not REST). Faster to set up, cheaper, CORS built-in. Attach to all Lambdas.

🗄️ DynamoDB

Single table design. On-demand pricing. No schema migrations. Fast to query.

🪣 S3

Audio file uploads. Presigned URLs for direct browser upload. Triggers Transcribe job.

🤖 OpenAI GPT-4o

Transcript parsing, vagueness scoring, recommendation generation. The brain of the system.

🎙️ AWS Transcribe

Speech-to-text for audio uploads. Supports speaker diarization (who said what).

⚛️ React + Vite

Frontend. Fast dev server. TailwindCSS or plain CSS. Axios for API calls.

🐙 GitHub API

Fetch commits per author per date. Free with personal access token. No OAuth needed for demo.

📌 Jira REST API

Fetch ticket status and changelog. Basic auth with API token. Works on Jira Cloud free tier.

📦 Node.js Libraries

axios, @aws-sdk/client-dynamodb, @aws-sdk/client-s3, @aws-sdk/client-transcribe, openai

🎨 React Libraries

recharts (charts), axios, react-router-dom, date-fns. Keep it minimal.

🔧 Dev Tools

Postman for API testing. AWS Console for infra. VS Code. No local AWS setup needed.

🤖 OpenAI Prompts

Exact prompts to use — copy paste these into your Lambda functions

Prompt 1 — Parse Transcript into Structured Standups

const prompt = ` You are an AI assistant that parses standup meeting transcripts. Extract each person's standup update from the transcript below. Team members: ${teamMembers.join(', ')} Date: ${date} Transcript: ${transcript} Return a JSON array with this exact structure: [ { "name": "Full Name", "email": "email@company.com", "yesterday": "what they did yesterday", "today": "what they plan to do today", "blockers": "any blockers or None" } ] Rules: - If someone didn't speak, omit them - Keep yesterday/today/blockers concise but accurate - If blockers not mentioned, set to "None" - Return ONLY valid JSON, no explanation `;

Prompt 2 — Score Vagueness of a Standup Update

const prompt = ` Rate the specificity of this standup update on a scale of 0 to 1. 0 = extremely vague (e.g. "working on stuff", "making progress") 1 = extremely specific (e.g. "fixed bug in auth middleware, PR #123 raised") Standup update: Yesterday: ${yesterday} Today: ${today} Blockers: ${blockers} Return ONLY a JSON object: { "vaguenessScore": 0.0-1.0, "vagueReasons": ["reason1", "reason2"] } `;

Prompt 3 — Generate Scrum Master Recommendation

const prompt = ` You are an experienced scrum master. Based on the data below, give a short, actionable recommendation for this team member. Be empathetic — the goal is to help, not blame. Team member: ${name} Sprint health score: ${healthScore}/100 Risk level: ${riskLevel} Flags detected: ${flags.map(f => `- ${f.type}: ${f.message}`).join('\n')} Git commits this week: ${gitCommits} Jira ticket moved: ${jiraTicketMoved} Days reporting same task: ${stalenessCount} Return ONLY a JSON object: { "recommendation": "2-3 sentence actionable recommendation", "urgency": "immediate | this-week | monitor", "suggestedAction": "1:1 | team-discussion | no-action" } `;

🤖 Microsoft Teams Bot Integration
Automatically capture standup transcripts without anyone doing anything manually
💡 Teams provides meeting transcripts automatically if transcription is enabled. We can pull these via the Microsoft Graph API — no bot joining calls needed.
Option 1 — Microsoft Graph API (Recommended ✅)
1
Register an Azure AD App
Go to portal.azure.com → Azure Active Directory → App Registrations → New Registration. Name it "SprintPulse AI". Note the Client ID and Tenant ID.
2
Grant API Permissions
In your app → API Permissions → Add: OnlineMeetings.Read.All and CallRecords.Read.All. Grant admin consent. These allow reading meeting transcripts.
3
Create Client Secret
App → Certificates & Secrets → New Client Secret. Copy the value immediately — store in Lambda env vars as TEAMS_CLIENT_SECRET.
4
Enable Transcription in Teams
Teams Admin Center → Meetings → Meeting Policies → Enable "Transcription". This must be on for transcripts to be available via API.
5
Fetch Transcript via Graph API
After standup call ends, call: GET /communications/callRecords/{callId}/sessions then GET /communications/callRecords/{callId}/sessions/{sessionId}/segments to get the transcript text per speaker.
6
Set Up Webhook (Auto-trigger)
Register a subscription: POST /subscriptions with resource communications/callRecords. Teams will POST to your Lambda URL every time a meeting ends — fully automatic, no manual action needed.
Option 2 — Teams Bot (More Control)
1
Create Bot in Azure Bot Service
Azure Portal → Create Resource → Azure Bot. Connect to Teams channel. This gives you a bot that can be added to your standup meeting.
2
Bot Joins Meeting as Participant
Use POST /communications/calls Graph API to have the bot join the standup meeting. The bot listens but doesn't speak.
3
Receive Real-time Transcript
Subscribe to transcriptUpdated events. Teams sends transcript chunks in real-time as people speak. Bot collects and sends to SprintPulse analysis engine when meeting ends.
4
Post Summary to Teams Channel
After analysis, bot posts a card to your standup channel showing each person's health score and flags. Team sees results immediately after the call.
Option 3 — Slack Bot (If using Slack)
1
Create Slack App at api.slack.com
Create app → Add slash command /standup pointing to your Lambda URL. Team members type /standup and fill a modal form.
2
Daily Reminder Bot
Use Slack Workflow Builder or a scheduled Lambda (EventBridge) to DM each team member at 9am with a standup form. They fill it in Slack — no separate tool needed.
3
Post Analysis to Channel
After collecting all standups, bot posts a sprint health summary card to the team channel with scores and flags. Scrum master gets a DM for at-risk members.
Recommended Approach for Hackathon
✅ Graph API Webhook
Fully automatic. Meeting ends → webhook fires → transcript fetched → analysis runs. Zero manual steps. Best for demo.

RECOMMENDED
⚠️ Teams Bot
More impressive visually but needs Azure Bot Service setup and admin approval. Risky for 36-hour hackathon.

COMPLEX
💬 Slack Bot
Easiest to set up. Great if your team uses Slack. Slash command + daily reminder covers most use cases.
